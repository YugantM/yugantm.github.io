Introduction

The "textcleaner" library contains several different types of components.

It contains some built-in functions to handle and operate text-data. It may also handle numbers but doing so is not advisable. The library have some dependencies which includes some other open-source libraries like NLTK,REGEX and default library string.

Some modules are set to the private and bounded through configuration to limit its uasage. These modules are defined to help other functions communicate with each other. Though you can use other modules freely which are mentioned in the documentation under built-in functions. All those functions can be called individually or in combinations.

Development of this library is not funded by any organization. This project is created and maintained by individuals who want to contribute to the open-source community.
Kindly visit this link pypi.org/project/textcleaner.

Installation

For windows & Linux:
pip install textcleaner


For updating:
pip install textcleaner --upgrade


Before importing:

If you have not downloaded the stopwords from nltk it will ask you to download while importing it.
To avoid that, write following in the command prompt.

python
>>> import nltk
>>> nltk.download('stopwords')

Aim is to remove dependecies as much as possible in the later versions.

Usage:
>>> import textcleaner as tc #or
>>> import textcleaner #or
>>> from textcleaner import *
>>> data = tc.document(<file_name>)
Notes:
  • data is an object of document class. data is used as an example for the documentation.
  • inplace argument is defined to set the nature of the operation of the object.
    Default argument is inplace = False which means before the operation, the function will generate a copy of the object, return the replica and keeping original data unchanged.
    The other option is inplace = True which means do the operation on the object and return it.
  • token_it function is an exception for chain calling because it returns tokens which is not supported datatype for the functions.

clear_blank_lines( )
clear_blank_lines is capable of removing all the blank lines from the text-data.
#definition: clear_blank_lines(self,inplace=false)
y = tc.clear_blank_lines(<file_name>)
#or
y = data.clear_blank_lines()
sample_text:

This is a sample text


with blank lines in it. Call the function and remove them.

Code:
>>> import textcleaner as tc
>>> data = tc.document(sample_text)
>>> data.clear_blank_lines()
Output:

This is a sample text
with blank lines in it. Call the function and remove them.


lemming( )
lemming executes lemmatization. Lemmatization usually refers to doing things properly with the use of a vocabulary and morphological analysis of words, normally aiming to remove inflectional endings only and to return the base or dictionary form of a word, which is known as the lemma. [source]

lemming will convert words to lemma with the help of the vocabulary. This library uses lemmatization method from NLTK.

#definition: lemming(self,inplace=false)
y = tc.lemming(<file_name>)
#or
y = data.lemming()
sample_text:

Root words are great.
It helps to summarize the articles or something.

Code:
>>> import textcleaner as tc
>>> tc.document(sample_text).lemming() #chain calling
Output:

Root word are great.
It help to summarize the article or something.

you can see conversions like: words to word, helps to help, articles to article

lower_all( )
lower_all sets all the characters of the text-data to lowercase. Which is desirable in most scenarios of text-data cleaning and furthur processing.
#definition: lower_all(self,inplace=false)
y = tc.lower_all(<file_name>)
#or
y = data.lower_all()
sample_text:

In SoMe Cases we MighT geT the data Like tHis.

Code:
>>> import textcleaner as tc
>>> tc.document(sample_text).lower_all()
Output:

in some cases we might get the data like this.


main_cleaner( )
main_cleaner implements all the cleaning methods in one call. It may take a bit to execute but processes data in a manner where user can pass the text data and it will return a fully cleaned and consumable data.

This function has one editional argument op. Default value of op is 'sents' which means the function will return list of sentences. Other value is 'words' which means the function will return a flat list of words from the data.

main_cleaner does not implement token_it.
#definition: main_cleaner(self,op='sents',inplace=false)
y = tc.main_cleaner(<file_name>)
#or
y = data.main_cleaner()
sample_text:

This is a SamPle tExt.WiTh blank spaces      , With symbols like %^(#,


with blank lInes iN it. CaLl the fuNctions aNd reMoVe them.

Code:
>>> import textcleaner as tc
>>> data = tc.document(sample_text)
>>> data.main_cleaner()
Output:

sample text with blank space symbol like
blank line call function remove

#definition: clear_blank_lines(self,inplace=false)

stemming()
stemming uses SnowBall Stemmer from NLTK library to convert words to their respective root word. [source]
#definition: clear_blank_lines(self,inplace=false)
y = tc.stemming(<file_name>)
#or
y = data.stemming()
strip_all( )
strip_all strips off '.\n' from each line of the file. Moreover it also clears unnecessary whitespace from the text as well with help of REGEX.

It also has one argument x with default value of '.\n' which can be changed as per the user's requirement.
#definition: clear_blank_lines(self,x='.\n',inplace=false)
y = tc.strip_all(<file_name>)
#or
y = data.strip_all()
token_it( )
token_it tokenizes the data. It is an inplace operation by default and can not be changed by argument. These tokens are stored into class in different variable named 'tokens'. [source]
#definition: clear_blank_lines(self,inplace=false)
y = tc.clear_blank_lines(<file_name>)
#or
y = data.clear_blank_lines()
y.tokens
